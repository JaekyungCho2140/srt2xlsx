1. 개요

1.1 프로그램 소개

Subtitle File Converter는 다국어 자막 파일(.srt)과 Excel 파일(.xlsx) 간 양방향 변환을 지원하는 Windows 전용 데스크톱 도구입니다.

주요 기능

  • SRT → XLSX 변환: 다중 언어 SRT 파일을 하나의 Excel 파일로 통합
  • XLSX → SRT 변환: Excel 파일에서 언어별 SRT 파일 생성
  • 엄격한 검증: 타임스탬프 동기화, 자막 번호 순서 자동 검증
  • 자동 인코딩 감지: UTF-8, CP949, 다양한 인코딩 자동 처리
  • 상세 오류 리포트 (v1.7.0 신규): 파일명, 자막 번호, 라인 번호를 포함한 정확한 오류 위치 표시
  • LLM 자막 검증 (v1.7.0 신규): LLM이 생성한 자막 파일의 오류를 빠르고 정확하게 검증

1.2 시스템 요구사항

최소 요구사항
  • 운영체제: Windows 10 (64-bit) 이상
  • RAM: 4GB 이상 (8GB 권장)
  • 디스플레이: 1366×768 이상
  • 저장 공간: 100MB

2. 설치 및 실행

2.1 프로그램 설치

Note
이 프로그램은 포터블 방식으로 별도의 설치 과정이 필요 없습니다.
  1. srt2xlsx.exe 파일을 원하는 폴더에 복사
  2. 더블클릭으로 바로 실행
  3. 관리자 권한 불필요

2.2 첫 실행

프로그램을 처음 실행하면 다음과 같은 창이 나타납니다:

  • 창 크기: 300×250 픽셀 (고정)
  • 초기 모드: SRT → XLSX (마지막 사용 모드 기억)
  • 창 위치: 화면 중앙 (마지막 위치 기억)

Step 3. 변환할 SRT 파일 선택

5. 오류 해결

🆕 v1.7.0 신규 기능: 상세 오류 리포트

이제 오류 발생 시 정확한 파일명, 자막 번호, 라인 번호를 포함한 상세 정보가 별도 창에 표시됩니다!

표시 정보:
  • 어떤 파일에서 오류가 발생했는지
  • 몇 번째 자막에 문제가 있는지
  • 원본 파일의 정확한 라인 번호
  • 오류에 대한 명확한 설명
추가 기능:
  • 클립보드 복사: 오류 내용을 복사하여 공유 가능
  • 자동 로그 저장: validation_errors_YYYYMMDD_HHMMSS.log 파일 자동 생성

5.1 오류 케이스별 상세 안내

❌ 케이스 1: 자막 번호 누락

문제 파일 예시:

1
00:00:01,000 --> 00:00:02,000
첫 번째 자막

2
00:00:03,000 --> 00:00:04,000
두 번째 자막

4  ← 3번이 누락됨
00:00:05,000 --> 00:00:06,000
네 번째 자막

v1.7.0 오류 메시지:

=== 파일: subtitle_KO.srt ===
  • 파일: subtitle_KO.srt | 자막 #3
    자막 번호 누락 (3를 기대했으나 4가 발견됨)

총 1개 파일에서 1개 오류 발견

해결 방법:

  1. 오류 메시지에서 누락된 자막 번호 확인 (예: 자막 #3)
  2. 해당 파일을 텍스트 에디터로 열기
  3. 누락된 번호의 자막 블록 추가 또는 번호 재정렬

❌ 케이스 2: 중복된 자막 번호

문제 파일 예시:

1
00:00:01,000 --> 00:00:02,000
첫 번째 자막

42
00:00:05,000 --> 00:00:06,000
마흔두 번째 자막

43
00:00:07,000 --> 00:00:08,000
마흔세 번째 자막

42  ← 42번이 중복됨
00:00:09,000 --> 00:00:10,000
중복된 자막

v1.7.0 오류 메시지:

=== 파일: subtitle_KO.srt ===
  • 파일: subtitle_KO.srt | 자막 #42
    중복된 자막 번호 (42번이 라인 9과 17에 나타남)

총 1개 파일에서 1개 오류 발견

해결 방법:

  1. 오류 메시지에서 중복된 자막 번호와 라인 번호 확인
  2. 텍스트 에디터에서 표시된 라인으로 이동 (예: 라인 9, 17)
  3. 중복된 자막 중 하나를 삭제하거나 번호 수정

❌ 케이스 3: 잘못된 타임스탬프 형식

문제 파일 예시:

15
00:01:65,000 --> 00:01:67,000  ← 초가 60 이상 (잘못됨)
열다섯 번째 자막

v1.7.0 오류 메시지:

=== 파일: subtitle_EN.srt ===
  • 파일: subtitle_EN.srt | 자막 #15 | 라인 6
    타임스탬프 형식 오류: '00:01:65,000' (초는 60 미만이어야 함)

총 1개 파일에서 1개 오류 발견

해결 방법:

  1. 오류 메시지에서 자막 번호와 라인 번호 확인
  2. 잘못된 타임스탬프 값 확인 (예: 00:01:65,000)
  3. 올바른 형식으로 수정 (예: 00:02:05,000)
  4. 타임스탬프 규칙: HH:MM:SS,mmm (시:분:초,밀리초)
  5. 초와 분은 반드시 60 미만이어야 함

❌ 케이스 4: 파일 간 타임스탬프 동기화 오류

원인: 동일 번호의 자막이 언어별로 다른 타임스탬프를 가짐

문제 예시:

subtitle_KO.srt:
3
00:05:30,000 --> 00:05:32,500
세 번째 자막

subtitle_EN.srt:
3
00:05:30,100 --> 00:05:32,600  ← KO와 0.1초 차이
Third subtitle

v1.7.0 오류 메시지:

=== 파일 간 검증 ===
  • 자막 #3
    파일 간 타임스탬프 불일치:
  - subtitle_KO.srt: 00:05:30,000 --> 00:05:32,500
  - subtitle_EN.srt: 00:05:30,100 --> 00:05:32,600

총 1개 파일에서 1개 오류 발견

해결 방법:

  1. 오류 메시지에서 자막 번호 확인 (예: 자막 #3)
  2. 두 파일의 타임스탬프 비교
  3. 한 파일의 타임스탬프를 다른 파일과 동일하게 수정
  4. 주의: 1ms라도 다르면 오류 발생

❌ 케이스 5: 자막 개수 불일치

원인: 각 언어 파일의 자막 블록 개수가 서로 다름

문제 예시:

subtitle_KO.srt: 120개 자막
subtitle_EN.srt: 118개 자막  ← 2개 부족

v1.7.0 오류 메시지:

=== 파일 간 검증 ===
  • 파일 간 자막 개수 불일치:
  - subtitle_KO.srt: 120개
  - subtitle_EN.srt: 118개

총 1개 파일에서 1개 오류 발견

해결 방법:

  1. 오류 메시지에서 각 파일의 자막 개수 확인
  2. 개수가 적은 파일에서 누락된 자막 찾기
  3. 누락된 자막 블록 추가 또는 불필요한 자막 삭제

❌ 케이스 6: 여러 오류 동시 발생

특징: v1.7.0부터 모든 오류를 한 번에 수집하여 표시

문제 파일 예시:

1
00:00:01,000 --> 00:00:02,000
첫 번째 자막

4  ← 3번 누락
00:00:05,000 --> 00:00:06,000
네 번째 자막

5
00:00:07,000 --> 00:00:08,000
다섯 번째 자막

5  ← 5번 중복
00:00:09,000 --> 00:00:10,000
중복된 자막

6
00:02:75,000 --> 00:02:77,500  ← 초가 60 이상
여섯 번째 자막

v1.7.0 오류 메시지:

=== 파일: subtitle_KO.srt ===
  • 파일: subtitle_KO.srt | 자막 #3
    자막 번호 누락 (3를 기대했으나 4가 발견됨)
  • 파일: subtitle_KO.srt | 자막 #5
    중복된 자막 번호 (5번이 라인 13과 17에 나타남)
  • 파일: subtitle_KO.srt | 자막 #6 | 라인 22
    타임스탬프 형식 오류: '00:02:75,000' (초는 60 미만이어야 함)

총 1개 파일에서 3개 오류 발견

해결 방법:

  1. 오류 목록의 각 항목을 순서대로 확인
  2. 각 오류의 위치 정보를 사용하여 해당 위치로 이동
  3. 모든 오류를 수정한 후 재변환
💡 장점
첫 번째 오류만 보고 수정 → 재시도 → 두 번째 오류 발견의 반복 과정이 사라집니다!
한 번에 모든 문제를 확인하고 수정할 수 있습니다.

5.3 v1.7.0 오류 창 사용 방법

🪟 오류 상세 창 기능

창 크기: 600×400 픽셀 (스크롤 가능)

표시 내용:
  • 파일별 그룹핑된 오류 목록
  • 각 오류의 정확한 위치 (파일명, 자막 번호, 라인 번호)
  • 명확한 오류 설명
기능:
  • 클립보드에 복사: 오류 내용을 복사하여 메모장 등에 붙여넣기 가능
  • 닫기: 오류 창을 닫고 파일 수정 작업 시작

클립보드 복사 기능 활용

💡 Tip
오류 내용을 팀원과 공유하거나 기록으로 남기고 싶을 때:
  1. 오류 창에서 "클립보드에 복사" 버튼 클릭
  2. 메모장, Slack, 이메일 등에 붙여넣기
  3. 버튼이 "✓ 복사됨"으로 1초간 변경되어 복사 완료 확인

자동 로그 파일

오류 발생 시 프로그램 실행 파일과 같은 폴더에 자동으로 로그 파일이 생성됩니다:

validation_errors_20251021_140530.log

용도:

  • 나중에 참고하기 위한 기록 보관
  • 자동화 테스트 및 분석
  • 오류 패턴 분석

  • 오류가 있으면 상세 오류 창 표시
↓
3단계: 오류 수정
  • 오류 창에서 정확한 위치 확인

7. FAQ

Q1. 한 번에 몇 개의 파일을 변환할 수 있나요?

A. 이론적으로는 제한이 없으나, 10,000행 이하를 권장합니다. 대용량 파일은 처리 시간이 길어질 수 있습니다.

Q2. 프로그램 설정은 어디에 저장되나요?

A. settings.ini 파일이 프로그램 실행 파일과 같은 폴더에 자동 생성됩니다. 마지막 사용 모드, 창 위치, 마지막 폴더 정보가 저장됩니다.

Q7. v1.7.0의 오류 로그 파일은 어디서 확인하나요? (신규)

A. 프로그램 실행 파일(srt2xlsx.exe)과 같은 폴더에 validation_errors_YYYYMMDD_HHMMSS.log 형식으로 저장됩니다. UTF-8 인코딩으로 메모장에서 열람 가능합니다.
